# tests/meridian — the Meridian estate suite

Validates the MERIDIAN demo corpus (`fixtures/meridian`, generated by
`ontoforge.estates.meridian_gen`, seed 7) and the product claim it exists for:
the **generic engine** answers CEO-grade questions over a messy 10-table
enterprise estate it has never seen, with atom-level citations.

| file | what it proves |
|---|---|
| `test_generator.py` | byte-determinism (committed fixtures == seed-7 output), <8 MB, valid GTIN-14 / ISO 6346 check digits, no accidental candidate-key hijackers, gold answers computed (never hardcoded) |
| `test_overlap.py` | cross-system entity overlap floors (>=60% of purchasing suppliers in 2+ systems, heroes in 3+), deliberate orphans, the battery-swelling text bridge |
| `test_warts.py` | every documented wart class present at bounded rates with hero rows clean (zero-stripped vendor ids, name variants, kg/lb + SF/SQM mixes, locale dates, null-token zoo, re-keyed double entries, mojibake, dangling FKs, the KPC trap) |
| `test_gate.py` | **the gate** (slow): generic pipeline over `fixtures/meridian` answers >=7/9 answerable gold questions fully correct WITH citations; both unanswerables abstain; the trick-unit question is rejected by the type checker; zero confidently-wrong |
| `test_demo_cli.py` | `ontoforge demo` argument surface |

Regenerate the corpus: `uv run python scripts/build_meridian_corpus.py`.
If the gate regresses, improve question phrasing realism or generator overlap
in `meridian_gen.py` — never hardcode engine answers.
